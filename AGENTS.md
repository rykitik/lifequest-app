# AGENTS.md

## Идентичность проекта

LifeQuest — это mobile-first персональная операционная система жизни.

Это не generic todo app. Не превращай проект в Trello, Notion, Habitica, Jira или обычный фитнес-трекер.

Продукт должен ощущаться как:
- спокойная персональная панель управления;
- оператор в духе JARVIS;
- геймифицированная база жизни;
- лёгкая система планирования;
- система восстановления после плохих дней.

## Критические продуктовые принципы

1. Mobile-first.
2. Один главный шаг в каждой секции экрана.
3. Никаких перегруженных dashboard-экранов.
4. На главном экране не показывать длинные списки задач.
5. Планирование должно быть легче, чем прокрастинация.
6. Приложение должно снижать когнитивную нагрузку.
7. Возврат в систему важнее идеального выполнения.
8. Ядро-компаньон поддерживает, а не оценивает.
9. Не использовать стыд, наказания или агрессивные streak-механики.
10. В MVP делать Центр промптов вместо платной внутренней AI-интеграции.

Весь пользовательский интерфейс должен быть на русском языке.

## Правила AI / Центра промптов

В MVP не вызывать OpenAI API.

Реализовать Центр промптов:
- карточки промптов;
- сбор контекста;
- предпросмотр сгенерированного промпта;
- кнопка «Скопировать промпт»;
- кнопка «Открыть ChatGPT».

Карточки промптов:
- Помоги распланировать день
- Я залип, верни меня в систему
- Мне тревожно, помоги мыслить ясно
- Деньги / финансы
- Отношения / одиночество
- Разобрать мою жизнь

Сгенерированный prompt должен включать:
- текущее состояние;
- главный квест;
- быструю победу;
- запасной план;
- релевантные цели;
- запрос пользователя;
- желаемый формат ответа.

## Правила Companion Core

Companion Core — не питомец и не мультяшный персонаж.

Он должен быть:
- светящимся ядром;
- спокойным;
- ненавязчивым;
- визуально живым;
- связанным с прогрессом пользователя.

Состояния:
- idle / спокоен
- focused / сфокусирован
- low_energy / мало энергии
- overloaded / перегружен
- recovering / восстанавливается
- evolving / эволюционирует

Для MVP использовать Framer Motion. Не делать тяжёлый 3D без явного запроса.

## UX-правила

Экран «Сегодня»:
- Ядро-компаньон сверху;
- выбор состояния;
- главный квест;
- быстрая победа;
- запасной план;
- быстрые действия.

Экран «План»:
- быстрый ввод задачи;
- chips классификации;
- маршрут дня;
- развернуть задачу на 3 шага.

Экран «Тело»:
- вес;
- вода;
- шаги;
- тренировка;
- питание.

Экран «Деньги»:
- баланс;
- долг;
- ежедневные денежные действия;
- спокойная финансовая сводка.

Экран «Ядро»:
- XP;
- уровень;
- прогресс секторов;
- эволюция Companion;

Rescue:
- пользователь выбирает проблему;
- приложение даёт одно короткое действие;
- награда — Recovery XP.

## Не делать

Не делать:
- социальные функции;
- leaderboard;
- встроенную AI-интеграцию через API;
- сложный инвентарь;
- 20 задач на главном экране;
- shame-based streaks;
- романтического, манипулятивного или needy-ассистента;
- Telegram как основное приложение;
- desktop-first UI.
