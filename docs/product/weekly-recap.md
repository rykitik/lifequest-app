# LifeQuest Weekly Recap

Weekly Recap — короткое локальное отражение недели на `/core`.

Его задача: показать, какие жизненные модули получили сигналы, какие вехи появились, какой контур стоит держать мягким фокусом дальше и что выбрать направлением следующей недели.

Это не длинный отчёт, не календарная heatmap и не оценка дисциплины.

## Отличие от AI Weekly Review

Local Weekly Recap:

- строится полностью локально;
- использует только агрегаты из stores;
- не вызывает backend, sync или OpenAI API;
- показывает короткий системный вывод.

AI Weekly Review:

- остаётся глубоким внешним разбором через Prompt Center;
- запускается только по инициативе пользователя;
- не меняет JSON contract Prompt Center;
- может быть сохранён в Weekly Review History.

Weekly Recap может показать hint к AI Weekly Review, но не подменяет его и не открывает внешние сервисы автоматически.

## Локальные сигналы

Тело:

- дни с телесными сигналами;
- вода, шаги, движение и чек-ин только агрегатно;
- без точного веса и медицинских деталей.

Деньги:

- создана ли база;
- был ли импорт или сверка;
- есть ли предупреждение;
- без сумм, балансов, last4, операций, merchant names и rawDescription.

Фокус:

- выполнен ли Daily Quest;
- есть ли завершённые route/quick win сигналы;
- без raw названий личных задач.

Восстановление:

- low/drifted mode;
- recovery XP;
- recovery route;
- weekly risk как факт, без raw текста.

Система:

- onboarding;
- backup;
- заполненность профиля;
- system milestones.

Companion:

- customization;
- evolution level;
- companion milestones.

## Статусы недели

- `empty` — Неделя ещё собирает сигналы.
- `forming` — База недели формируется.
- `active` — Неделя дала устойчивые сигналы.
- `strong` — Неделя хорошо закреплена.

Не используются формулировки вроде «плохая неделя», «провал» или «слабая дисциплина».

## Strongest Module

Strongest module выбирается по безопасным weekly signals:

- агрегаты активности;
- completed daily/route signals;
- milestones текущей недели;
- локальные system/companion сигналы.

Если сигналов нет, модуль не объявляется усиленным.

## Attention Module

Attention module — это мягкий фокус, а не слабое место.

Правила:

- при явном low energy, drifted mode или recovery risk выбирается `recovery`;
- без recovery risk выбирается модуль с наименьшим количеством weekly signals;
- tie-breaker deterministic и идёт по порядку модулей Skill Tree;
- copy всегда спокойная: «стоит держать мягким фокусом».

## Next Week Focus

Фокус следующей недели строится из attention module.

Примеры:

- Тело → собрать телесный сигнал;
- Деньги → уточнить финансовую базу;
- Фокус → закрепить главный квест;
- Восстановление → стабилизировать восстановление;
- Система → защитить локальную базу;
- Companion → настроить связь с Core.

## Privacy

Weekly Recap не выводит:

- банковские операции;
- суммы расходов;
- балансы;
- last4;
- merchant names;
- rawDescription;
- точный вес;
- медицинские детали;
- raw task title;
- raw prompt text;
- imported AI answer text.

Разрешены только безопасные агрегаты:

- телесный сигнал;
- финансовое обновление;
- квест дня выполнен;
- веха открыта;
- backup создан;
- профиль настроен.

## Anti-Toxicity

Запрещены:

- «неделя провалена»;
- «ты ничего не сделал»;
- «плохая дисциплина»;
- «серия потеряна»;
- «верни серию»;
- «потерян прогресс»;
- наказания;
- leaderboard/social/casino rewards.

Тон:

- «Неделя ещё собирает сигналы.»
- «Можно начать с одного мягкого шага.»
- «Система продолжит с текущей точки.»
- «Модулю нужен бережный сигнал.»
