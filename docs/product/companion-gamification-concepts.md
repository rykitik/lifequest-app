# Companion gamification concepts

## Concept A: System Core / JARVIS Dashboard

Фокус: серьёзный sci-fi интерфейс, Companion Core как центральный системный модуль, минимум RPG-терминов. Прогресс называется стабильностью системы.

### Today screen

- Companion Core сверху.
- Состояние дня: мало энергии, стабильно, в ресурсе, дрейф.
- Один главный квест.
- Одна быстрая победа.
- Запасной план.
- Быстрые действия: начать 2 минуты, Rescue, Центр промптов.

### Profile/System screen

- Система: уровень 7.
- Стабильность: 74%.
- Core state: сфокусирован.
- Модули: тело, деньги, фокус, восстановление, порядок.
- Следующая форма: Core Mk-III.

### Companion evolution block

- Сейчас: Сфокусирован.
- Сигнал собран, 74% до следующей формы.
- Живое ядро с аурой и модулями.
- Формы показываются как режимы системы, не как кнопки.

### Progress block

- Стабильность системы.
- Рост модулей.
- Последняя полезная фиксация.
- Recovery XP виден как возврат в систему, не как слабость.

### Daily action loop

Открыл приложение -> выбрал состояние -> сделал один шаг -> система усилила модуль -> Core дал спокойную обратную связь.

### Reward feedback

"Шаг засчитан. Фокус +12 XP. Система стабильнее."

### 360px mobile

Один главный блок на экран. Сектора не показываются большой таблицей, а раскрываются по одному. Evolution block использует 2 колонки или горизонтальный список.

### Pros

- Самый premium-fit для LifeQuest.
- Меньше риска детскости.
- Хорошо поддерживает идею personal OS.

### Cons

- Может быть слишком холодным, если Companion не получает эмоциональную роль.
- RPG-ощущение слабее.

### Complexity

Средняя. Нужны новые тексты, профильная сводка и переработка блока эволюции.

### Risk of overgamification

Низкий.

### Match with LifeQuest

Очень высокий.

## Concept B: Life RPG Profile

Фокус: сильнее ощущение игры, профиль персонажа, уровень, навыки, ветки развития, достижения и companion evolution.

### Today screen

- Профильный статус сверху: уровень, класс, Core form.
- Главный квест дня.
- 1-3 meaningful actions.
- Навыки, которые усилятся после действия.

### Profile/System screen

- Профиль: Ваня.
- Уровень 7.
- Класс: Архитектор системы.
- Навыки: тело, деньги, фокус, воля, восстановление.
- Companion: Core Mk-II.
- Вехи развития.

### Companion evolution block

- Core Mk-I -> Core Mk-II -> Core Mk-III.
- Открытые формы и будущие формы.
- Состояния Core показаны как режимы внутри формы.

### Progress block

- Навыки с уровнями.
- Значимые вехи.
- Мягкие достижения: "вернулся после тяжёлого дня", "закрыл базу".

### Daily action loop

Выбрал маршрут -> выполнил квест -> усилил навык -> получил спокойный feedback -> приблизил Core evolution.

### Reward feedback

"Фокус усилен. До Core Mk-III осталось 26%."

### 360px mobile

Профиль должен быть компактным: avatar/core, уровень, 2-3 главных навыка. Остальное раскрывается ниже.

### Pros

- Сильное ощущение "я прокачиваю себя".
- Легко объяснить прогресс.
- Хорошо поддерживает competence.

### Cons

- Риск уйти в Habitica-like стиль.
- Нужно строго избегать монстров, наказаний, перегруза бейджами.

### Complexity

Средне-высокая. Нужна модель профиля и навыков.

### Risk of overgamification

Средний.

### Match with LifeQuest

Высокий, если визуально оставить JARVIS/System Core.

## Concept C: Calm Companion OS

Фокус: Companion как партнёр. Меньше цифр, больше состояния, заботы и направления. Прогресс виден мягко.

### Today screen

- Живое Ядро сверху.
- Короткое состояние: "Сегодня бережём базу".
- Один мягкий следующий шаг.
- Rescue доступен без ощущения провала.

### Profile/System screen

- Мой ритм.
- Что стало стабильнее.
- Где нужна поддержка.
- Companion growth через историю восстановления и малых побед.

### Companion evolution block

- "Ядро стало стабильнее".
- Следующий мягкий шаг.
- Формы описаны не технически, а эмоционально: спокойный, собранный, восстановление, эволюция.

### Progress block

- Меньше чисел.
- Больше понятных итогов недели.
- XP остаётся вторичным слоем.

### Daily action loop

Понял состояние -> сделал минимальный шаг -> получил поддержку -> система сохранила возвращение.

### Reward feedback

"Ты укрепил базу. Следующий шаг можно сделать позже."

### 360px mobile

Очень хорошо: меньше таблиц и карточек, больше воздуха.

### Pros

- Лучший вариант для восстановления после плохих дней.
- Снижает давление.
- Сильная relatedness с Companion.

### Cons

- Может не дать достаточно "игрового профиля".
- Пользователю сложнее увидеть прокачку.

### Complexity

Средняя. Главная работа в копирайтинге, IA и мягком progress UI.

### Risk of overgamification

Низкий.

### Match with LifeQuest

Высокий, но не закрывает весь запрос на "мой профиль / прокачка".

## Recommendation: Hybrid A + B

Рекомендуется Hybrid A + B.

Визуально: JARVIS / System Core.  
Мотивационно: RPG profile / прокачка.  
Эмоционально: Companion растёт вместе с пользователем.

Это даёт LifeQuest взрослую sci-fi оболочку, но сохраняет понятное ощущение роста: уровень, модули, форма Core, следующая эволюция.

## Что внедрять первым

1. Player/System Profile screen: имя, уровень, статус системы, Core form, модули.
2. Новый Companion evolution block: текущая форма, следующая форма, прогресс, будущие формы.
3. Skills/modules: тело, деньги, фокус, восстановление, порядок.
4. Reward feedback, который явно связывает действие с модулем и стабильностью.

## MVP System Profile

Первый MVP System Profile развивается внутри существующего раздела "Ядро". Это сохраняет навигацию компактной: Today остаётся экраном действия, а Ядро становится экраном идентичности и прокачки.

MVP показывает:

- профиль пользователя;
- уровень системы;
- Core form;
- состояние Companion;
- ветки развития;
- безопасные recent milestones;
- один следующий полезный шаг.

Подробное описание: [system-profile.md](./system-profile.md).

## Что отложить

- Кастомизацию Core.
- Сложные ветки развития.
- Большой achievement system.
- Social/multi-user.
- Backend sync.
