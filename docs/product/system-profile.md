# System Profile

System Profile — главный экран ощущения "это моя система и моя прокачка". В текущей навигации он живёт в существующем разделе "Ядро", потому что этот пункт уже связан с Companion Core, уровнем и прогрессом.

## Зачем он нужен

Today отвечает на вопрос "что сделать сейчас". System Profile отвечает на вопрос "кто я в LifeQuest и как растёт моя база".

Экран должен быстро показывать:

- имя и роль пользователя;
- уровень системы;
- текущую форму Companion Core;
- прогресс до следующей формы;
- ветки развития;
- последние безопасные вехи;
- один следующий полезный шаг.

## Какие данные показывает MVP

MVP собирается только из local-first stores:

- `useSettingsStore`: имя, роль, onboarding, backup metadata;
- `useProgressStore`: уровень, XP, daily summary, sectors;
- `useCompanionStore`: состояние, сообщение, stability score;
- `useBodyStore`: сегодняшний чек-ин и локальные body logs;
- `useMoneyStore`: факт денежной базы, импорт, баланс-чек, предупреждения без деталей операций;
- `useWeeklyReviewStore`: факт сохранённого недельного обзора;
- `useTodayStore` + `todayNextStep`: следующий полезный шаг.

Экран не требует аккаунта, backend, sync или OpenAI API.

## Почему это не Habitica

System Profile использует игровые элементы только как язык компетентности:

- уровень системы;
- форма Core;
- ветки развития;
- вехи.

Он не использует:

- питомца;
- монстров;
- наказания;
- потерю серии;
- лидерборды;
- сравнение с другими;
- случайные casino-награды.

Пользователь не "проигрывает день". Он видит, какие сигналы уже есть и какой один шаг поможет системе стать устойчивее.

## Как Companion связан с прогрессом

Companion Core показывает:

- текущую форму, например `Core Mk-II`;
- текущее состояние: спокоен, сфокусирован, восстановление и так далее;
- прогресс до следующей формы;
- короткую системную фразу.

Форма Core — это не персонаж и не питомец. Это визуальный системный модуль, который становится сильнее по мере накопления стабильных сигналов.

## Empty states

Новый пользователь не должен видеть пустоту или обвинение. Если данных мало, экран говорит:

- "База собирается";
- "Нужно больше сигналов";
- "Первые вехи появятся после чек-ина, импорта или недельного обзора";
- "Companion Core активен, но ждёт первых данных".

## Безопасность данных

Recent milestones не показывают:

- суммы;
- названия операций;
- медицинские детали;
- raw descriptions из импортов;
- личные заметки.

Вехи формулируются системно: "Резервная копия создана", "Чек-ин тела выполнен", "Финансовый импорт принят".
