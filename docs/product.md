# Спецификация продукта LifeQuest

## Тезис продукта

LifeQuest — персональная операционная система жизни.

Приложение помогает:

- планировать без перегруза;
- начинать с малого;
- возвращаться после тяжёлых дней;
- снижать прокрастинацию;
- отслеживать тело и деньги;
- видеть визуальный рост;
- использовать ChatGPT внешне через готовые промпты.

## Позиционирование

LifeQuest остаётся solo-first продуктом, но теперь проектируется как `local-first now, multi-user ready later`.

Это означает:

- сейчас основной сценарий — один пользователь на одном устройстве без аккаунта;
- позже появится account mode и синхронизация;
- migration local data в аккаунт должна быть предусмотрена заранее, но не должна ломать текущий UX.

## Режимы продукта

### Local mode

- всё работает без регистрации;
- данные хранятся локально;
- backup/export-import нужен для сохранения состояния;
- local-first опыт остаётся главным UX MVP.

### Account mode

- позже появятся `login/register`;
- данные будут связаны с `userId`;
- backend будет синхронизировать состояние между устройствами;
- локальные данные станут cache/offline-слоем.

### Migration

- пользователь сможет создать аккаунт позже;
- local data и backup можно будет перенести в аккаунт;
- перенос должен быть мягким и не требовать повторного ручного ввода всего прогресса.

## Пользовательский контекст

Продукт проектируется вокруг пользователя, который:

- перегружается от сложного планирования;
- уходит в избегание, видео, игры и другие быстрые циклы;
- хочет больше контроля над работой, телом, деньгами и привычками;
- любит футуристичные интерфейсы и спокойный JARVIS-like стиль;
- нуждается в структуре без стыда и давления.

## Ядро-компаньон

Роль:

- эмоциональный якорь;
- индикатор прогресса;
- спокойное присутствие ассистента.

Он не:

- питомец;
- человек;
- романтический персонаж;
- терапевт;
- судья.

## Спасательный режим

Цель:

помочь пользователю вернуться к действию, когда система застряла.

Результат:

- одно короткое действие;
- Recovery XP;
- мягкий возврат в систему.

## Центр промптов

В MVP нет внутреннего OpenAI API.

Центр промптов:

- собирает контекст из приложения;
- генерирует prompt на русском языке;
- даёт скопировать его;
- открывает внешний ChatGPT.

## Критерии успеха

Приложение успешно, если пользователь:

- быстро понимает следующий шаг;
- не тонет в длинных списках;
- чаще возвращается в систему после плохих дней;
- чувствует поддержку без осуждения;
- может сохранить свой прогресс даже без backend через backup.
