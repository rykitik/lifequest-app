# Roadmap LifeQuest

LifeQuest уже прошёл стадию статичного UI-скелета. Сейчас основной вектор — сделать local-first систему реально полезной в ежедневном и недельном цикле без перегруза и без обязательного аккаунта.

## Текущий сильный цикл

1. Пользователь ведёт тело через быстрый чек-ин.
2. Пользователь задаёт старт учёта денег и обновляет операции через выписку Сбера.
3. LifeQuest собирает недельный контекст по телу, деньгам, фокусу и восстановлению.
4. Внешний ChatGPT возвращает разбор.
5. Пользователь выбирает, какие действия применить.
6. LifeQuest сохраняет недельный итог локально.

Этот цикл уже полезнее обычного трекера, потому что связывает состояние тела, деньги, энергию и фокус.

## Ближайшие приоритеты

### 1. Полный просмотр недельного итога

Сейчас история недель есть, но карточка показывает только краткое содержание. Следующий удобный слой — открыть сохранённый недельный итог целиком без повторного обращения к ChatGPT.

Важно:

- не делать тяжёлый dashboard;
- показать summary, bodyFocus, moneyFocus, risk, coreMessage и выбранные actions;
- сохранить приватность: без полного prompt, выписок и полного списка операций.

### 2. Улучшить ежедневный вход в систему

Главный экран уже показывает маршрут, но после появления денег и weekly review ему нужен более умный “следующий шаг”.

Идея:

- если нет body check-in сегодня — предложить один телесный шаг;
- если давно не обновлялись деньги — предложить обновить выписку или сверить баланс;
- если есть сохранённый weekly risk — мягко учитывать его в сегодняшнем запасном плане.

### 3. Деньги: сверка и уверенность

Money module уже умеет baseline, import, debt и correction. Следующий слой должен улучшать доверие к цифрам.

Полезно:

- отдельный список последних корректировок;
- понятное объяснение, почему баланс отличается от выписки;
- быстрый фильтр imported/manual/adjustment;
- не добавлять сложную бухгалтерию и банковскую интеграцию.

### 4. Body: недельная динамика без calorie tracker

Body не должен превращаться в медицинское приложение или счётчик калорий.

Полезный минимум:

- простая недельная сводка воды, шагов, движения и питания;
- мягкий статус “данных мало / достаточно”;
- связь с weekly review без диагнозов и жёстких оценок.

### 5. Backup как спокойный safety net

До полноценного sync backup остаётся самым важным способом не потерять local-first данные.

Полезно:

- заметнее показать дату последнего backup;
- предложить backup после крупных изменений: импорт денег, сохранение недельного итога, настройка baseline;
- не делать автоматическую отправку наружу.

## Среднесрочно

### Account mode и sync

Auth и первый settings sync уже есть, но общий sync runtime ещё не готов. Его стоит делать только после того, как local-first UX станет устойчивым.

Порядок:

1. sync только для одного безопасного домена;
2. idempotency и очередь изменений;
3. conflict UI;
4. migration local data в аккаунт;
5. только потом расширять на quests/progress/body/money.

### Больше банков и источников

Сейчас поддержан локальный импорт Сбера. Расширять стоит осторожно:

- сначала улучшить надёжность Сбер PDF/text;
- затем добавить ручной CSV/text импорт как универсальный fallback;
- не делать bank auth, scraping и backend upload.

## Что не является ближайшим приоритетом

- социальные функции;
- leaderboard;
- полноценный calorie tracker;
- инвестиции и сложные кредитные расчёты;
- автоматический bank sync;
- обязательная авторизация;
- большой dashboard;
- встроенный OpenAI API.
